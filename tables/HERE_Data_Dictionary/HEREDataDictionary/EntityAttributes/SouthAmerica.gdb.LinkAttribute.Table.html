1 OBJECTID Object ID ESRI unique ID. OID 4  
2 LINK_ID Link ID Unique identifier for the Link to which the Street Address relates. Link ID is used to identify each link in the database. Integer 4  
3 FOUR_WHEEL_DRIVE Four-wheel Drive Identifies roads which are suitable only for vehicles with Four-Wheel drive. Four-Wheel Drive is primarily used for cartographic representation (map display). This attribute can also affect routing algorithms by assigning higher penalties to Four-Wheel drive roads. String 1 LinkAttribute.FOUR_WHEEL_DRIVE
4 SCENIC_ROUTE Scenic Route Scenic Route is a route offering pleasing views of natural landscapes. Scenic Route can be used for route calculation, prioritising scenic routes and for map display. String 1 LinkAttribute.SCENIC_ROUTE
5 PARKING_LOT_ROAD Parking Lot Road Identifies roads internal to parking lot areas. Parking Lot Road can be used to enable special guidance in Parking Lot areas and provides enhanced map display. String 1 LinkAttribute.PARKING_LOT_ROAD
6 PRIORITY_ROAD Priority Road Priority Road defines road stretches that have signs indicating priority on the road. On these roads all traffic has priority over the traffic on the incoming roads. Priority Road can be used for route planning to achieve lower fuel consumption. By prioritising priority roads during route planning, the “stop and go” at intersections, resulting in high fuel consumption, can be minimized. String 1 LinkAttribute.PRIORITY_ROAD
7 CARPOOL_ROAD Carpool Road The Carpool Road attribute identifies a Link where at some point all lanes serve as Carpool lane. Carpool Road can be used to avoid Carpool links in the routing, and can be used for explicit instructions (guidance) in relation to Carpool links. String 1 LinkAttribute.CARPOOL_ROAD
8 REVERSIBLE Reversible The Reversible attribute identifies that a Link is fully reversible, which implies that traffic is allowed in both directions of the same lane, at specific times of day. Note: Current specifications provide coding for Reversible Links, however the lane counting is duplicated for reversible links. The double lane counting for reversible links is sub-optimal for display related lane-level content like Lane Marking. This specification provides an enhanced model to resolve the lane counting issue for reversible links. String 1 LinkAttribute.REVERSIBLE
9 EXPRESS_LANE Express Lane The Express Lane attribute identifies links that serve as express lane. Express Lane can be used to prioritise ongoing traffic to take the express lane. String 1 LinkAttribute.EXPRESS_LANE
10 TRANSITION_AREA Transition Area Transition Area defines the stretch of road where the number of lanes changes and lane markings are not present on the road. Transition Area can be used for display purposes to inform the user that the number of lanes changes and show when the transition starts and ends. String 1 LinkAttribute.TRANSITION_AREA
11 EXPANDED_INCLUSION Expanded Inclusion Identifies cartographic features or links that meet the expanded inclusion criteria. This attribute makes all of the links or cartographic features meeting the expanded inclusion criteria easily identifiable. It allows for the option to not publish these links/ features if the inclusion causes size issues. Can be used to differentiate links included only for limited use from those published with limited use attribution. For example, when a portion of a hiking trail shares geometry with an unpaved road for use by autos, only those links dedicated for hiking are indicated for limited use only. Note: Features that exceed the standard size requirement do not receive Expanded Inclusion. However, features that are flagged as Expanded Inclusion at a lower Coverage Indicator level retain the Expanded Inclusion flag when the Coverage Indicator gets upgraded to a higher level where the Standard Inclusion size overlaps with that of the Expanded Inclusion. For example, if a lake feature flagged with Expanded Inclusion in Canada has a Coverage Indicator value of N4 and is upgraded to N0, it maintains the Expanded Inclusion flag regardless of meeting standard inclusion size for Coverage Indicator value = N0. If the Expanded Inclusion attribute does not apply to the link or cartographic feature, the attribute is not published. Prior to the Expanded Inclusion initiative, Bridges and Tunnels were only included in the database if they are at least 200 metres in length. Expanded Inclusion allows for Bridges and Tunnels between 5 and 200 metres to be included (if they are significant). These newly included Bridges and Tunnels will have the Expanded Inclusion attribute applied. SmallInteger 2 LinkAttribute.EXPANDED_INCLUSION
12 SUPPLEMENTAL_GEO_BITSET Supplemental Geometry Bitset Provides a classification for road geometry types included for special or limited use. These geometries are provided in addition to HERE’s standard road network. Supplemental Geometry Bitset can be used for map display and route calculation (avoidance of Supplemental Geometry). Integer 4  
13 DELIVERY_ROAD Delivery Road A Delivery Road is a road leading from a main path to the loading/unloading area for a business. Delivery Road can be used for route calculation, route guidance, and map display. String 1  
14 TRUCK_ROAD_TYPE Truck Road Type Defines the Truck Road Types as defined by NORMA 12 specification. SmallInteger 2
