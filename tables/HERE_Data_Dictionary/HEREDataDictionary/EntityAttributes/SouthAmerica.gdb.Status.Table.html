2 STATUS_ID Status ID The Status ID is the Key into the STATUS table, which is used to publish various attributes that provide information on the status of the navigable link (e.g., Detailed City, Urban, In Process Data or Transport Verified). See 5.31 Status (Status) section. Integer 4  
3 IS_ALIGNED Is Aligned Indicates that this node is on the boundary between two database regions and that its X and Y Coordinates have been matched to a node in the adjacent region. Identifies nodes at the boundary of a database for merging with other databases. String 1 Status.IS_ALIGNED
4 STUB_LINK Stub Link Stub Link identifies navigable Links that intersect the included long haul navigable network. String 1 Status.STUB_LINK
5 LONG_HAUL_NAV Long Haul Navigable Long Haul Navigable indicates that a navigable Link is part of a Long Haul product. String 1 Status.LONG_HAUL_NAV
6 FULL_GEOMETRY Full Geometry Full Geometry identifies that a link has the complete geometry of the road network surrounding it. Note: The same value is applied to all of the links comprising a countryΓÇÖs road network. Thus, Full Geometry indicates that a link is within a country that has all of the road network geometry included.Full Geometry allows more precise route calculation and route guidance. It also allows more accurate timing of guidance. String 1 Status.FULL_GEOMETRY
7 IN_PROCESS_DATA In Process Data In Process Data identifies that a link is included prior to completion to full specification. The In Process Data attribute on links indicates that the links do not have full routing attributes. Travel Direction, Speed Category, etc. may not represent reality. String 1 Status.IN_PROCESS_DATA
8 URBAN Urban This attribute identifies links located within a Built-up Area polygon. This attribute can be used for map display. String 1 Status.URBAN
9 TRANSPORT_VERIFIED Transport Verified Transport Verified indicates if the link has been verified for the attribution as defined in the Trucks specification. Transport Verified attribute can be used to determine if the link can be reliably used for transport routing. String 1 Status.TRANSPORT_VERIFIED
10 ENHANCED_GEOMETRY Enhanced Geometry Enhanced Geometry indicates that the geometry associated with a Link meets the positional accuracy. Enhanced Geometry may be used to identify links with an enhanced level of geometrical accuracy. String 1 Status.ENHANCED_GEOMETRY
11 DETAILED_CITY Detailed City This attribute specifies whether a link is part of a Detailed City coverage area. If Detailed City = N, then the inclusion level may be either Network, In-Process Data, or City-to-City. This attribute allows systems to adapt to different levels of attributing. String 1 Status.DETAILED_CITY
