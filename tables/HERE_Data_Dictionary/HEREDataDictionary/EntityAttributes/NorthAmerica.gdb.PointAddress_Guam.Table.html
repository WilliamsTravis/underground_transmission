1 OBJECTID Object ID ESRI unique ID. OID 4

3 POINT_ADDRESS_ID Point Address ID The Point Address ID is a unique identifier for the Point Address. Integer 4  
4 LINK_ID Link ID Unique identifier for the Link to which the Street Address relates. Link ID is used to identify each link in the database. Integer 4  
5 STREET_ADDRESS_ID Street Address ID Unique identifier for a named, addressed navigable link. Integer 4  
6 ADDRESS Address The Address is the Address of the point address. String 150  
7 ADDRESS_TYPE Address Type Address Type identifies the type of the house number range assigned to the link. The Address Type is used in conjunction with related address attributes, and the associated name for destination selection and geocoding. String 1 PointAddress_Guam.ADDRESS_TYPE
8 LANGUAGE_CODE Language Code The language associated with the feature name. The Language Code, in conjunction with Prefix, Feature Name, Street Type, and Suffix, can be used in destination selection, route guidance, and map display. String 3 PointAddress_Guam.LANGUAGE_CODE
9 BUILDING_NAME Building Name Building Name is a Point Address attribute to publish the name of the building that identifies the Point Address. For certain countries the Building Name is an alternative to the address for identifying a Point Address, and required to define the Point Address. The Building Name attribute can be used for destination selection. The Building name combination provides an addressable entity. String 150  
10 SIDE Side The Side of the link where the POI is located. String 1 PointAddress_Guam.SIDE
11 ARRIVAL_LINK_ID Arrival Link ID Arrival Link is the link with the drive-to Link for the Point Address. The Arrival Link represents the link where one would like to guided to when selecting a specific Point Address. Note: There are a few cases where the Arrival Link is a pedestrian-only link. The Arrival Link is used for route calculation and route guidance, and represents the drive-to Link for the Point Address. Integer 4  
12 ARRIVAL_SIDE Arrival Side Arrival Side indicates on which side of the Arrival Link the Point Address is located. String 1 PointAddress_Guam.ARRIVAL_SIDE
13 DISPLAY_LAT Display Latitude The Display Latitude is the latitude of the display position of the POI. Double 8  
14 DISPLAY_LON Display Longitude The Display Longitude is the longitude of the display position of the POI. Double 8  
15 ENHANCED Enhanced Enhanced is an attribute to identify if the Point Feature has been verified or has been automatically generated. The Enhanced attribute can be used to differentiate between Point Addresses that are verified, and Point Addresses that are automatically generated but do not necessarily represent a verified position. String 1 PointAddress_Guam.ENHANCED
16 ISO_COUNTRY_CODE ISO Country Code A three-letter country code as defined by the International Organization for Standardization. The ISO Country Code indicates in which country the feature is located. Different layers describe different features. ISO Country Code attribution is applied to Navigable Links, Administrative Hierarchies, and POIs in various layers. String 3 PointAddress_Guam.ISO_COUNTRY_CODE
17 ST_NAME Street Name Street Name is the Full Street Name of the Link. This is the preferred street name for the Link. Street Name can also be a Route Number, Exit Number, or Junction Name, if such a name is the preferred Street Name. String 255  
18 ST_NM_PREF Street Name Prefix Street Name Prefix is a directional identifier that precedes the Street Name of a road. The Street Name Prefix is included in the full name for roads. The full name should be used for route guidance and destination selection instead of individual components Street Name Prefix, Street Type, Street Name, and Street Name Suffix. If the full name is used for display or destination input list the Street Name Prefix should not be displayed after the name or Street Type, this may allow the Street Name Prefix to be confused with the Street Name Suffix. String 255  
19 ST_TYP_BEF Street Type Before Street Type Before indicates if the Street Type is before or after the Base Name. Street Type Before can be used to determine the placement of the Street Type for usage in destination selection, map display and route guidance. String 255  
20 ST_NM_BASE Street Name Base Street Name Base is the Base Name of the Street Name. String 255  
21 ST_NM_SUFF Street Name Suffix A Street Name Suffix is a directional identifier that follows, but is not included in, the Street Name of a road. The Street Name Suffix is included in the full name for roads. The full name should be used for route guidance and destination selection instead of individual components Street Name Prefix, Street Type, Street Name, and Street Name Suffix. If the full name is used for display or destination input list the Street Name Prefix should not be displayed after the name or Street Type, this may allow the Street Name Prefix to be confused with the Street Name Suffix. String 255  
22 ST_TYP_AFT Street Type After Street Type After is the Street Type of the Street Name. It is only published when the Street Type appears after the Base Name. See 5.58.8 Street Type Before (Streets). String 255  
23 ST_TYP_ATT Street Type Attachment Street Type Attached indicates whether the Street Type is attached or separated by a space from the Base Name. Street Type Attached can be used to determine the location of the Street Type for usage in destination selection, map display and route guidance. String 255  
24 ZIP ZIP ZIP Code. String 255  
25 LATITUDE Latitude Latitude. Double 8  
26 LONGITUDE Longitude Longitude. Double 8  
27 CITY CITY City. String 255  
28 STATE STATE State abbreviation for the state within which the applicable postal code falls. String 255  
29 COUNTY County USPS county name within which the applicable postal code falls. String 255  
30 STATE_NAME State Name State Name. String 255
