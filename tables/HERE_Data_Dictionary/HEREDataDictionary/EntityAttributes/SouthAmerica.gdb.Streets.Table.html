5 STREETNAMELNG Street Name Lng Street Name Language Code is the Language Code that is applicable to the Street Name. The Language Code in the Streets layer represents the ISO639-1 language code (e.g., ‘en’) which is a two character language code. Only cultural-neutral langauge codes are published. HERE does not publish culturally-specific language codes like, for example, “en-US’ or ‘nl-NL’. String 5 Streets.StreetNameLng
6 STREETNAME Street Name The Street Name attribute is the full spelling of the street name including the Street Name Prefix, Street Name Suffix and Street Type. String 100  
7 STREETNAMEPREFIX Street Name Prefix Street Name Prefix is a directional identifier that precedes the Street Name of a road. The Street Name Prefix is included in the full name for roads. The full name should be used for route guidance and destination selection instead of individual components Street Name Prefix, Street Type, Street Name, and Street Name Suffix. If the full name is used for display or destination input list the Street Name Prefix should not be displayed after the name or Street Type, this may allow the Street Name Prefix to be confused with the Street Name Suffix. String 10  
8 STREETTYPEBEFORE Street Type Before Street Type Before provides the Street Type of the Street Name, but it is only published when the Street Type appears before the Street Name. Street Type Before can be used to determine the placement of the Street Type for usage in destination selection, map display and route guidance. String 30  
9 STREETNAMEBASE Street Name Base Street Name Base is the Base Name of the Street Name. String 60  
10 STREETTYPEAFTER Street Type After Street Type After is the Street Type of the Street Name. It is only published when the Street Type appears after the Base Name. See 5.58.8 Street Type Before (Streets). String 30  
11 STREETNAMESUFFIX Street Name Suffix A Street Name Suffix is a directional identifier that follows, but is not included in, the Street Name of a road. The Street Name Suffix is included in the full name for roads. The full name should be used for route guidance and destination selection instead of individual components Street Name Prefix, Street Type, Street Name, and Street Name Suffix. If the full name is used for display or destination input list the Street Name Prefix should not be displayed after the name or Street Type, this may allow the Street Name Prefix to be confused with the Street Name Suffix. String 10  
12 DIRONSIGN Direction On Sign The official directional identifiers assigned to highways. Direction on Sign should be used in conjunction with Feature Name, Name on Road Sign, and Name Route Type. For route guidance, US-101 South should be used and not just US-101 when appropriate. String 10  
13 METERS Meters Meters indicates the length of the Link in meters. Integer 4  
14 SPEEDCAT SpeedCat Speed Category classifies the general speed trend of a road based on posted or legal speed and is provided to enhance route calculation and the timing of route guidance. Speed Category values represent the combination of several factors besides legal speed limit (e.g., physical restrictions or access characteristics). Therefore Speed Category values can differ from the From/To Reference Speed Limit values, which represent the legal speed limit only. Speed Category may be used to estimate link traversal times, to prioritise link selection during route calculation, and to calculate timing of the route guidance. SmallInteger 2 Streets.SpeedCat
15 FUNCCLASS Functional Class Functional Class defines the arterial classification (hierarchical network) used to determine a logical and efficient route for a traveller. Functional Class can be used to determine sets of links that form connected graphs. SmallInteger 2 Streets.FuncClass
16 DIRTRAVEL Direction Of Travel Direction of Travel identifies legal travel directions for a navigable link. Direction of Travel enables correct route calculation, route guidance, and map display. For example, display of one-way icon. String 1 Streets.DirTravel
17 CONTRACC ContrAcc Controlled Access identifies roads with limited entrances and exits that allow uninterrupted high speed traffic flow. These roads constitute the interstate/freeway network in the U.S. and the motorway network in Europe. The Controlled Access attribute can be used for map display, avoidance of freeway/motorway, and route guidance timing. String 1 Streets.ContrAcc
18 FERRYTYPE FerryType An attribute that defines whether a link is or is not a ferry link. If the link is a ferry link, Ferry Type will describe if it is a boat ferry link, or a rail ferry link. It is a combination of the Boat Ferry and Rail Ferry attributes. See Table 5-97 below for Boat Ferry and Rail Ferry combinations that define the Ferry Type. String 1 Streets.FerryType
19 RAMP Ramp Ramps are connectors that provide access between roads that do not cross at grade. This attribute allows explication of manoeuvres involving ramps (e.g. “Take the ramp”). This attribute can also be used for display at different zoom levels or different colors and for route guidance when determining if sign text should be used. String 1 Streets.Ramp
20 ROUNDABOUT Roundabout A roundabout is a contiguous loop with consistent one-way traffic throughout the circle that controls the traffic flow from converging roads. This attribute allows explication of manoeuvres involving roundabouts (i.e. “Take the third exit from roundabout”). String 1 Streets.Roundabout
21 PAVED Paved Describes roads that are made of materials which create a solid surface. Paved is primarily used for cartographic representation (map display). This attribute could also affect routing algorithms by assigning higher penalties to unpaved roads. String 1 Streets.Paved
22 PUBLICACCESS Public Access The Public Access attribute indicates whether or not the Link allows public access. Route Calculation: The Public Access attribute indicates if the road element can be used in route calculation to avoid specific navigable links. Route Guidance: The Public Access attribute can be used in route guidance for additional guidance instructions. Map Display: The Public Access attribute can be used in map display to differentiate navigable links. String 1 Streets.PublicAccess
23 TOLLWAY Tollway This attribute identifies a link for which a fee must be paid to use the road. Tollway identifies roads that require payments for traversal. This attribute’s intended uses are display and map publishing purposes. This attribute used in conjunction with the Toll Structure condition may be used for toll avoidance routing. String 1 Streets.Tollway
24 TIME_ZONE Time Zone Time Zone indicates the GMT time zone in which the Link is located. Time Zone can be used to identify the time zone applicable to an administrative area. String 4  
25 DST_EXIST DST Exist Indicates if the Link is located in an administrative area where Daylight Saving Time (DST) is observed. String 1 Streets.DST_Exist
26 TIMEZONEID Time Zone ID Not published by HERE. Integer 4

38 ISOCOUNTRYCODE ISO Country Code A three-letter country code as defined by the International Organization for Standardization. The ISO Country Code indicates in which country the feature is located. Different layers describe different features. ISO Country Code attribution is applied to Navigable Links, Administrative Hierarchies, and POIs in various layers. String 3 Streets.ISOCountryCode
39 CARPOOLROAD CarpoolRoad The Carpool Road attribute identifies a Link where, at specific times, all lanes serve as carpool lanes. Carpool Road can be used to avoid Carpool links in the routing, and for explicit instructions (guidance) in relation to Carpool links. String 1 Streets.CarpoolRoad
40 EXPRESSLANE ExpressLane The Express Lane attribute identifies Links that serve as express lanes. Express Lane can be used to optimize the current route by taking the express lane. String 1 Streets.ExpressLane
