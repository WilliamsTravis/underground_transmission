12 BRIDGE Bridge Bridge indicates if the Link is a bridge. A bridge is a structure that allows a road, railroad or walkway to pass over another road, railroad, water feature, or valley.This attribute can be used for display or route guidance. String 1 Link.BRIDGE
13 TUNNEL Tunnel Tunnel indicates if the Link is a tunnel. A tunnel is a covered passageway through or This attribute can be used for display or route guidance under an obstruction. String 1 Link.TUNNEL

15 FUNCTIONAL_CLASS Functional Class Functional Class defines a hierarchical network used to determine a logical and efficient route for a traveller. Functional Class can be used to determine sets of links that form connected graphs. SmallInteger 2 Link.FUNCTIONAL_CLASS
16 CONTROLLED_ACCESS Controlled Access Controlled Access identifies roads with limited entrances and exits that allow uninterrupted high speed traffic flow. These roads constitute the interstate/freeway network in the U.S. and the motorway network in Europe. The Controlled Access attribute can be used for map display, avoidance of freeway/motorway, and route guidance timing. String 1 Link.CONTROLLED_ACCESS
17 TRAVEL_DIRECTION Travel Direction Travel Direction identifies legal travel directions for a navigable link. Travel Direction enables correct route calculation, route guidance, and map display. For example, display of one-way icon. String 1 Link.TRAVEL_DIRECTION
18 BOAT_FERRY Boat Ferry A Boat Ferry represents the generalised route of a ferry for passengers or vehicles over water. This attribute can be used for map display. String 1 Link.BOAT_FERRY
19 RAIL_FERRY Rail Ferry A Rail Ferry represents the generalised route of a ferry for passenger vehicles via rail. This attribute can be used for map display. String 1 Link.RAIL_FERRY
20 MULTI_DIGITIZED Multi-Digitized The Multiply Digitised attribute identifies links that are digitised with one line per direction of traffic instead of one line per road and using the rules listed below. For wide roads a link is digitised for each direction of traffic (instead of for each road) in order to improve map matching of the car to the road map. The Multiply Digitised attribute identifies these opposing lanes of traffic when it is reasonable to represent them as a single line on displays and printed maps. When route guidance calls for a u-turn involving a multi-digitised road, the guidance iven should be “make a u-turn at intersection” and not “make a left turn”, followed by “make a left turn.”. String 1 Link.MULTI_DIGITIZED
21 DIVIDER Divider This attribute identifies the presence of a physical traffic blocking divider.Divider coding enables a system to prevent left turns (in right-side driving countries), right turns (in left-side driving countries), and U-turns at divided intersections and in the middle of divided roads. See diagrams in Specification section below. String 1 Link.DIVIDER
22 DIVIDER_TYPE Divider Type Divider Type indicates if a divider is a Legal or Phsyical divider. In combination with the Divider attribute, enables a system to prevent left turns (in rightside driving countries), right turns (in left side driving countries), and U-turns at divided intersections and in the middle of divided roads. Divider Type identifies physically passable dividers for emergency vehicle routing applications. A physical divider identifies a physical barrier that cannot be traversed. String 1 Link.DIVIDER_TYPE
23 FRONTAGE Frontage Frontage Roads (aka Service Roads) are local roads that run parallel to and usually contain the name(s) and addresses of a road with a higher traffic flow. When it is necessary to be guided along the Frontage, the flag enables navigation systems to provide an appropriate message, for example, “take the Frontage Road.” This attribute enables better route guidance, if this attribute is not taken into consideration, a system might say “you are approaching an intersection of many Main Streets. Take the one in the middle.” The Frontage flag indicates that although the name of the road is the same, the functionality is different. In this way, the strange explications can be avoided. String 1 Link.FRONTAGE
24 PAVED Paved Describes roads that are made of materials which create a solid surface. Paved is primarily used for cartographic representation (map display). This attribute could also affect routing algorithms by assigning higher penalties to unpaved roads. String 1 Link.PAVED
25 RAMP Ramp Ramps are connectors that provide access between roads that do not cross at grade. This attribute allows explication of manoeuvres involving ramps (e.g. “Take the ramp”). This attribute can also be used for display at different zoom levels or different colors and for route guidance when determining if sign text should be used. String 1 Link.RAMP
26 PRIVATE Private Identifies roads not maintained by an organization responsible for maintenance of public roads. Private allows for unique cartographic representation of roads that restrict public use. Private allows for unique cartographic representation of roads that restrict public use. This attribute could also affect routing algorithms by assigning higher penalties to unpaved roads. String 1 Link.PRIVATE
27 TOLLWAY Tollway This attribute identifies a link for which a fee must be paid to use the road. Tollway identifies roads that require payments for traversal. This attribute’s intended uses are display and map publishing purposes. This attribute used in conjunction with the Toll Structure condition may be used for toll avoidance routing. String 1 Link.TOLLWAY
28 POI_ACCESS POI Access POI Access (roads) connect Points of Interest (POIs) to the road network. These roads provide the only means of entrance or exit from a POI to a public road. POI Access attribute allows an entrance to be identified by the POI name for route guidance. For example, the entrance road to a golf course may be explicated as “Turn right at the golf course entrance.”. String 1 Link.POI_ACCESS
29 INTERSECTION_CATEGORY Intersection Catetory Intersection Category defines the type of intersection. It identifies links that are part of or associated with road intersections. These include links created when multiply digitised roads are intersected or when the centrelines of intersecting roads do not meet at a single node. Turn lanes, traffic islands, u-turn lanes, and ramps are also included in the intersection definition. Intersection coding can be used for explication and display. The flags indicate whether a driver will perceive a situation as one or multiple intersections. These flags imply that the situation may require different explication that what is implied by the geometry. String 1 Link.INTERSECTION_CATEGORY
30 SPEED_CATEGORY Speed Category Speed Category classifies the general speed trend of a road based on posted or legal speed and is provided to enhance route calculation and the timing of route guidance. Speed Category values represent the combination of several factors besides legal speed limit (e.g., physical restrictions or access characteristics). Therefore Speed Category values can differ from Speed Limit values, which represent the legal speed limit only. Speed Category may be used to estimate link traversal times, to prioritise link selection during route calculation, and to calculate timing of the route guidance. SmallInteger 2 Link.SPEED_CATEGORY
31 LANE_CATEGORY Lane Category Lane Category classifies a road based on the number of lanes in each direction. Lane Category can be used for cartographic representation of road widths on printed maps, as well as traffic management and display applications. Lane Category may also be used for route guidance timing. SmallInteger 2 Link.LANE_CATEGORY
32 COVERAGE_INDICATOR Coverage Indicator Coverage Indicator is an attribute to a navigable or non-navigable link that indicates a product level that corresponds to the inclusion of database features. It also indicates verification and completeness for a particular feature in the database. String 2 Link.COVERAGE_INDICATOR
33 CONFIDENCE_LEVEL_RATING Confidence Level Rating As of the Q3, 2016 product release, data for this attribute is no longer published. SmallInteger 2

35 TO_REF_NUM_LANES To Reference Number of Lanes This attribute indicates the number of lanes in the negative Link direction (Towards Reference Node). SmallInteger 2  
36 PHYSICAL_NUM_LANES PHYSICAL NUM LANES This attribute defines the actual number of lanes physically present on the link. The attribute is introduced for reversible road elements initially, but allows for adoption at non-reversible road elements in the future. Note: Reversible lanes are counted as one lane. Physical Number of Lanes can be used to correctly render the road layout, in combination with lane marking, for reversible links. SmallInteger 2

Speed Limit may be used to estimate link traversal times, to prioritise link selection during route calculation, to calculate timing of the route guidance, and for display purposes. SmallInteger 2  
38 TO_REF_SPEED_LIMIT To Reference Speed Limit From/To Speed Limit is the posted or legal speed limit in each travel direction on a road. Speed Limit may be used to estimate link traversal times, to prioritise link selection during route calculation, to calculate timing of the route guidance, and for display purposes. SmallInteger 2  
39 SPEED_LIMIT_SOURCE Speed Limit Source Defines the source used for the From and To Ref Speed Limit information. Speed Limit Source provides information to speed limit recognition applications about the generalised source of the speed limits in the database, to allow the application to choose between the speed limit from the database and the speed limit from the camera. String 1 Link.SPEED_LIMIT_SOURCE
40 LOW_MOBILITY Low Mobility Indicates if the driving condition on the link is considered low mobility. The Low Mobility attribute provides for display of links differently from other functional class 5 and low speed category roads. It further distinguishes functionality of functional class 5 roads similar to the Parking Lot Road and Four-wheel Drive attributes. This information can be used in route calculation to avoid specific links. SmallInteger 2 Link.LOW_MOBILITY
41 PUBLIC_ACCESS Public Access The Public Access attribute indicates whether or not the Link allows public access. Route Calculation: The Public Access attribute indicates if the road element can be used in route calculation to avoid specific navigable links. Route Guidance: The Public Access attribute can be used in route guidance for additional guidance instructions. Map Display: The Public Access attribute can be used in map display to differentiate navigable links. String 1 Link.PUBLIC_ACCESS

45 LIMITED_ACCESS_ROAD Limited Access Road Limited Access Road indicates separately digitised, high speed roads where traffic is controlled by ramps. The Limited Access Road attribute can be used for map display, avoidance of freeway/motorway, and route guidance timing. String 1 Link.LIMITED_ACCESS_ROAD
46 ROAD_CLASS Road Class Road Class identifies a road network based on governmental classification. Road Class can be used for map display. Additionally, Road Class can be used for route guidance, e.g., "Enter the express highway" or "Enter the urban expressway. SmallInteger 2
