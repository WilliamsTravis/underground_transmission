3 POI_ID POI ID Unique ID for the POI. Integer 4  
4 CAT_ID Category ID The Category ID defines the type of POI. String 4 POI.CAT_ID
5 ISO_COUNTRY_CODE ISO Country Code A three-letter country code as defined by the International Organization for Standardization. The ISO Country Code indicates in which country the feature is located. Different layers describe different features. ISO Country Code attribution is applied to Navigable Links, Administrative Hierarchies, and POIs in various layers. String 3 POI.ISO_COUNTRY_CODE
6 LANGUAGE_CODE Language Code The language associated with the feature name. The Language Code, in conjunction with Prefix, Feature Name, Street Type, and Suffix, can be used in destination selection, route guidance, and map display. String 3 POI.LANGUAGE_CODE
7 POI_NAME Poi Name The POI Name is the Preferred, Official Name of the POI. POI Names can be used for destination selection and map display for languages supported by HERE. String 200  
8 HOUSE_NUMBER House Number The House Number is the House Number of the POI. String 10  
9 FULL_HOUSE_NUMBER Full House Number The Full House Number is the Full POI House Number of the POI. String 35  
10 ACTUAL_ADDRESS Actual Address The Actual Address represents the full actual address of the POI but it is different from the street address where the entrance of the POI is physically located. Actual Address allows the actual address of a POI to be displayed when the POI entrance is not physically located at that address. String 200  
11 ACTUAL_CITY Actual City The Actual City is the parsed Actual City Name of the POI. String 50  
12 ACTUAL_POSTCODE Actual Postal Code The Actual Postal Code is the parsed Actual Postal Code of the POI. String 15  
13 ACTUAL_STREET_NAME Actual Street Name The Actual Street Name is the parsed Actual Street Name of the POI. String 75  
14 ACTUAL_HOUSE_NUMBER Actual House Number The Actual House Number is the parsed Actual House Number of the POI. String 10  
15 VANITY_CITY_ID Vanity City ID The Vanity City ID is the Admin Place ID representing the Vanity City of the POI. Key into the Admin table. Integer 4  
16 DISPLAY_ID Display ID The Display ID is a value which can be used for map display to render POIs in a logical manner. SmallInteger 2  
17 LINK_ID Link ID Unique identifier for the Link to which the Street Address relates. Link ID is used to identify each link in the database. Integer 4  
18 SIDE Side The Side of the link where the POI is located. String 1 POI.SIDE
19 STREET_ADDRESS_ID Street Address ID Unique identifier for a named, addressed navigable link. Integer 4  
20 ADMIN_PLACE_ID Admin Place ID The Admin Place ID is a unique identifier of the lowest administrative level in the hierarchy. Integer 4  
21 POSTAL_CODE Postal Code Provides the Postal Code area in which the POI is located.While this feature can be used for destination selection this is usually not recommended, except in the case of refined address resolution. For example, when the same address exists twice within a city, a postal code selection may narrow down which address is preferred. String 15  
22 PERCENT_FROM_REF Percent From Reference Node Percent From Reference Node is an attribute to a Point Of Interest (POI) and indicates the location of the POI on the link in terms of percent from the reference node. POI Percent from Reference Node can be used to determine icon placement and timing of guidance information. This is particularly useful when POIs do not have an address, the road does not carry addresses, or when the POI is located on a very long link. SmallInteger 2  
23 DISPLAY_LAT Display Latitude The Display Latitude is the latitude of the display position of the POI. Double 8  
24 DISPLAY_LON Display Longitude The Display Longitude is the longitude of the display position of the POI. Double 8  
25 NATIONAL_IMPORTANCE National Importance National Importance indicates if a POI is nationally important. Nationally important means that the POI name is recognisable without a city name. For example, Heathrow Airport is recognisable without the city name of London. For systems that require city input first, National Importance allows display of regionally prominent POIs in a scroll list without needing to identify the city name. String 1 POI.NATIONAL_IMPORTANCE
26 PRIVATE Private Identifies roads not maintained by an organization responsible for maintenance of public roads. Private allows for unique cartographic representation of roads that restrict public use. This attribute could also affect routing algorithms by assigning higher penalties to unpaved roads. String 1 POI.PRIVATE
27 CUISINE_TYPE Cuisine Type Defines the predominant Cuisine Type applicable to the restaurant POI by giving a textual description of the cuisine type. Implementation of Cuisine Type allows a user to select restaurants by desired food type. Cuisine Type enables restaurants to be indexed, queried, and displayed by cuisine type. String 35  
28 CHAIN_NAME Chain Name Chain Name defines the predominant Chain Name for the POI. String 35  
29 LOCATION_SCORE Location Score Location Score provides a percentage value indicating the confidence of location accuracy. The Location Score ranges in value from 0 - 100. SmallInteger 2  
30 PLACE_SCORE Place Score Place Score provides a percentage value indicating the completeness of POI attribution. The Place Score ranges in value from 0 - 100. SmallInteger 2  
31 CALCULATED_LEVEL Calculated Level Calculated Level indicates the overall level of completion and accuracy of a POI. The Calculated Level ranges from 1-4. String 1 POI.CALCULATED_LEVEL
32 ENTRANCE_TYPE Entrance Type Entrance Type identifies the type of entrance to a facility. Entrance Type can be used for destination selection and map display to identify preferred entrances to a POI. String 1  
33 IN_VICINITY In Vicinity The In Vicinity attribute identifies whether a POI has been associated with a road representing its true location, or on a road nearby due to Inter-Town/Network road inclusion levels. The attribute tells a system what kind of route guidance information should be provided to users. If the POI is not In Vicinity the system can state 'you have arrived'. If the POI is In vicinity, the system can tell the user the POI is nearby but that further routing advice is not available. String 1 POI.IN_VICINITY
34 DISPLAY_CLASS Display Class Display Class is a classification of features that enables a meaningful display of these specific features.The classification is assigned according to the importance of the associated feature. Display Class information can be used to determine which features to display at different zoom levels. Display Class information can be used to prune out icons/landmarks of lower importance for memory-limited devices. Note: Display Class definitions for hydro-graphic features use different criteria for individual classes than Display Class for Landmarks/POIs. SmallInteger 2 POI.DISPLAY_CLASS
