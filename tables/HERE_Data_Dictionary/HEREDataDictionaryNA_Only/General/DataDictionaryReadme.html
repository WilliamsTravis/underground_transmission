The HERE File Geodabase Plus Data Dictionary

This HTML based data dictionary describes HERE File Geodabase Plus geospatial data layers for North America. It was developed by HERE in order to:
  • Provide a streamlined yet comprehensive description of the File Geodabase Plus data product.
  • Provide a single access point, lightweight HTML front-end to the most essential information and metadata needed by an end user to rapidly assess the potential value of data for a particular purpose and get started using it.
  • Provide the details required to help end users easily and quickly make the best use of HERE data for creating map displays, classifications, labeling, filtering, reports, and graphs.
  • Ensure that users needing to create ArcMap .mxd documents have very clear information describing layer/table types, attribute data types, lengths, and allowed values, geodatabase domains, and relationships.
The information presented in this data dictionary was compiled and integrated primarily from HERE's File Geodatabase Reference Manual, as well as additional information about supplemental data from HERE technical customer support, HERE documentation, FGDC metadata, and the data itself.

Data Dictionary Organization

The main menu links at the top of each data dictionary HTML page include:
  • Read Me: Provides a basic overview of the data dictionary and an overall description of main links and content.
  • Overview: Describes the most essential and important aspects of File Geodabase Plus data.
  • Layers/Tables: Provides full descriptions of all File Geodabase Plus layers, tables, attributes, and lookup table values. From the main Layers/Tables page, users can drill down to review attribute descriptions. From a Layer or Table's Attribute page, users can drill down to review Lookup Table values.
  • Facility/Feature Finder: As a single File Geodabase Plus layer may contain multiple feature types, and the same feature type may be found in multiple layers, this table helps end users find the layers and attributes name=value pairs needed for defining specific feature based map displays, classifications, labels, and reports.
  • Relationships: The ESRI File Geodatabase style relationships among features and tables, including the direction and cardinality of the relationship.
  • Map Types: A list of all countries included in the File Geodatabase Plus data, the HERE region geodatabase the country is found in, and its HERE map type.
  • Admin Levels: A list of all administrative geography names used in HERE's multi-level area classification system.
  • Dictionary Reference: Provides detailed definitions of the data dictionary fields used to describe Layers, Tables, Attributes, and Lookup Tables.
Note: To enable HTML table sorting, browsers need to be set to allow script running.
